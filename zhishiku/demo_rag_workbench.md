# 公开演示资料：RAG 知识库工作台

这是一份可公开上传到本项目的脱敏 Markdown 示例文档，用于演示个人 RAG 知识库助手的核心流程。

## 本地 RAG 的目标

个人知识库助手会先把文档解析、切分并写入本地 Chroma 向量库。用户提问时，系统会从当前知识库中检索相关片段，再把片段作为上下文交给大模型生成回答。

这个流程适合整理课程笔记、项目文档、论文摘要、会议记录和学习资料。它强调“答案来自哪里”，而不是只给出一段无法追溯的模型输出。

## 混合检索

本项目的检索链路不只依赖向量相似度。它会结合向量检索、BM25、关键词候选、rerank 和 contextual compression。

BM25 适合处理精确词匹配，例如模块名、文件名、配置项、术语缩写。向量检索适合处理语义相近的问题。混合检索可以让中文资料问答在“精确命中”和“语义理解”之间取得更稳定的效果。

## 引用与证据

回答下方会展示引用来源，包括文件名、片段、分数和候选来源。这样可以判断答案是否真的来自当前知识库，也可以快速追踪原文。

如果问题明显超出资料范围，例如资料中没有提到某个实体或配置，系统应该简洁拒答，而不是整理无关内容。
