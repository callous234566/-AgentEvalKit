# 项目维护手册

本文档记录项目维护优先级、已完成维护项、待办项和执行规则。它比 README 更偏运维和后续开发决策。

## 一、维护原则

- 优先最小改动，不做无关重构。
- 依赖变更必须先做兼容性分析。
- RAG 链路变更必须验证上传、分块、检索、问答。
- 缓存变更必须验证失效路径。
- 文档、配置、代码必须同步更新。
- 不在真实知识库数据上做临时测试。

## 五、缓存维护规则

- 缓存 key 必须包含所有会影响行为的参数。
- 写入、删除、重命名后必须失效缓存。
- collection 快照只能作为服务生命周期内缓存，不替代 Chroma 持久化。
- 测试缓存逻辑时使用模拟 collection 或临时目录。
